MinGW

最新版 SDK: 使用 MinGW 的 MinGW, 不要用 TDM 的!

注意： 最新版本的CMake存在bug使得ogre在MinGW上构建不成功。 请使用早期版本的CMake（测试成成功的最新的版本是2.8.3）

早期的Ogre sdk: 你需要使用来自Twilight Dragon Media（TDM）的 MinGW中的GCC才能构建， 因为截止这篇文章完成时是tdm-mingw-1.908.0-4.4.1-2, 使用 GCC 4.4.1。早期版本的Ogre sdk 不能使用官方MinGW进行编译， 因为早期版本的Ogre sdk只能使用4.4.x的GCC才能进行编译。

将MinGW安装到 C:\mingw 然后将 C:\mingw\bin 添加到 PATH.

对于有开发经验的人来说， 他们更喜欢从源代码编译。 某些平台没有提供预编译的SDK， 也需要自己编译。从源代码编译的SDK更新， 同时允许开发者对自己或是ogre的代码进行各方面的调试。如果你想贡献补丁， 这也是必需的步骤。

注意：从版本 1.7 (Cthugha), Ogre 使用 CMake 作为主要的构建系统. 对于以前的版本， 参考 构建旧版 Ogre.

你可以从 Mercurial获取 最新稳定代码（适用于所有平台）. 如果你使用windows平台， 你可能还需要阅读 使用 TortoiseHG 获取源码

实际上的稳定版是某些特定的分支， 比如Cthugha的"v1-7"分支就是一个稳定版， 而不是Cthugha的主分支。主分支是最新的非稳定版， 并且和稳定版的API标准不完全兼容， 并且也有可能有错误， 只用于有经验的开发者。

Mercurial 简短指南

克隆 OGRE 当前稳定版本 (Cthugha) :

hg clone https://bitbucket.org/sinbad/ogre/ -u v1-7

克隆非稳定版 :

hg clone https://bitbucket.org/sinbad/ogre/

注意：你可以在分支之间随意切换， 使用下面的代码切换到稳定版

hg update v1-7
